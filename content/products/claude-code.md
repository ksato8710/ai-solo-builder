## Claude Codeとは？

Claude Codeは、Anthropicが開発した**ターミナルベースのAIコーディングエージェント**。GUIエディタではなく、**CLI（コマンドライン）でAIがコードを自律的に編集・実行**する新しいアプローチを取る。

従来のAIコード補完ツールが「人間の入力を待つ」のに対し、Claude Codeは「タスクを渡すと自分で考えて行動する」**エージェント型**。ファイルの読み書き、テストの実行、Gitコミット、エラー修正まで一気通貫で処理。

ターミナル派のソロビルダーにとって、**IDEを開かずにコーディングを完結**できる革命的なツール。

## 主要機能

### 自律的ファイル操作

プロジェクト内のファイルを読み込み、編集、作成、削除。「この関数をリファクタリングして」と言えば、関連ファイルを自動で特定して修正を適用。

### ターミナルコマンド実行

シェルコマンドを実行し、結果を解釈して次のアクションを決定。ビルド実行 → エラー確認 → 修正 → 再ビルドのループを自動化。

### Git統合

コミット、ブランチ作成、差分確認、マージなどのGit操作をAIが実行。コミットメッセージも自動生成。

### プロジェクト全体理解

CLAUDE.mdファイルでプロジェクトのコンテキストを伝えると、技術スタック、コーディング規約、アーキテクチャを理解した上で提案。

### マルチファイル編集

関連する複数ファイル（コンポーネント、テスト、型定義）を一括で修正。単一ファイルに閉じない複雑な変更を処理。

### エラー自動修復

ビルドエラー、テスト失敗を検出すると、原因を分析して自動修復を試行。人間の介入なしで問題解決。

## 使い方（Getting Started）

1. **インストール**: 
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```
2. **認証**: Anthropic APIキーを設定、またはMax契約でログイン
3. **プロジェクトに移動**: `cd your-project`
4. **起動**: `claude` コマンドで対話開始
5. **タスク指示**: 「新しいAPI endpointを追加して」など自然言語で指示

## CLAUDE.md設定

プロジェクトルートに`CLAUDE.md`を置くと、AIがプロジェクト固有のコンテキストを理解:

```markdown
# CLAUDE.md

## プロジェクト概要
Next.js 14のSaaSアプリ

## 技術スタック
- TypeScript, Tailwind CSS, Prisma, Supabase

## コーディング規約
- コンポーネント: src/components/
- 型定義: src/types/
- テスト: __tests__/
```

## FAQ

### Q: CursorやCopilotとの違いは？

A: Cursor/CopilotはGUIエディタ統合型。Claude CodeはCLI完結型のエージェント。ターミナルで自律的にタスクを実行する点が根本的に異なる。

### Q: APIキーとMaxプラン、どちらがいい？

A: ライトユースならAPI従量課金、ヘビーユースならMaxプラン（$100/月）。Max契約ならClaude Code含め追加コストなし。

### Q: 危険な操作（ファイル削除等）は止められる？

A: 設定で確認を入れることは可能。ただし、デフォルトでは高い権限で動作するため注意。重要ファイルはバージョン管理必須。

### Q: WindowsでもMac

でも使える？

A: macOS、Linux、Windows（WSL経由）で動作。ネイティブWindowsコマンドプロンプトは非対応。

### Q: Devinとの違いは？

A: Devinはブラウザ+VM環境で動作する「AI社員」。Claude Codeはローカル環境で動作するCLIツール。DevinはよりフルスタックでUI操作も可能、Claude Codeはコーディング特化。

### Q: 大規模プロジェクトでも使える？

A: 使えるが、コンテキストサイズ（トークン数）に注意。大きなプロジェクトはCLAUDE.mdでスコープを絞ることを推奨。

### Q: チーム開発に向いている？

A: Max Teamプランでチーム利用可能。ただし、各メンバーが個別に指示する形が基本。チーム協調機能はこれから。

## ソロビルダー向けの使いどころ

### 夜間バッチ処理的な開発

タスクを渡して就寝、朝起きたら完成（またはエラーログ）。人間の稼働時間外でも開発が進む。

### レガシーコードの解析・リファクタリング

巨大なコードベースを分析して「どこを触るべきか」を特定。段階的にリファクタリング。

### 反復的なタスク自動化

「全コンポーネントにError Boundaryを追加」「TypeScript strict mode対応」など、人間がやると面倒な作業を一括処理。

### SSH環境でのデバッグ

本番サーバーにSSH接続 → Claude Code起動 → その場で調査・修正。IDEのセットアップ不要。

## 注意点・制限

### 技術的な制限

- **ターミナル前提**: GUIでのコード確認は別途エディタが必要
- **トークン消費**: 大規模プロジェクトは高コスト
- **Windows**: WSL必須

### 運用上の注意

- **破壊的操作**: ファイル削除、上書きは慎重に（Git管理必須）
- **セキュリティ**: 本番環境での実行はリスクを理解した上で
- **コスト管理**: API従量課金の場合、複雑タスクは予算オーバー注意
