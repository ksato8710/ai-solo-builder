X（旧Twitter）のAPI利用を検討しているソロビルダーにとって、**現在の料金体系は最大の障壁**だ。

2022年の買収以降、APIアクセスは大幅に値上げされ、かつての「開発者フレンドリー」な時代は終わった。Free tierは実質「投稿専用」になり、データ取得には最低でも**月額$200**が必要になった。

しかし2025年10月、Xは**Pay-Per-Use Pricing Pilot**を発表。使った分だけ払う従量課金モデルの導入で、状況は少し変わりつつある。

この記事では、2026年2月時点の最新情報をもとに、X APIの全貌を解説する。

## 現行の料金プラン（2026年2月時点）

### 料金比較表

| プラン | 月額 | 投稿（Write） | 取得（Read） | 用途 |
|--------|------|---------------|--------------|------|
| **Free** | $0 | 1,500件/月 | **0件** | 投稿専用ボット |
| **Basic** | $200 | 50,000件/月 | 15,000件/月 | 趣味・小規模 |
| **Pro** | $5,000 | 300,000件/月 | 1,000,000件/月 | 商用利用 |

### Free Tier — 実質「投稿専用」

Free tierは**データ取得（Read）がほぼ不可能**。

- 投稿: 1,500件/月（ユーザーレベル）
- 取得: **100件/月**（アプリレベル）— 実質使えない
- App ID: 1つ

**用途:** 単純な自動投稿ボットのみ。メンション監視、ハッシュタグ追跡、トレンド分析は**不可能**。

### Basic Tier — $200/月

最低限のデータ取得が可能になる最初の有料プラン。

- 投稿: 50,000件/月
- 取得: 15,000件/月
- App ID: 2つ
- API: v1.1 & v2 両対応

**15,000件は多いのか？**

実は**非常に少ない**。ブランドキーワードを数個監視するだけで、数日で使い切る可能性がある。リアルタイム監視やトレンド分析には到底足りない。

**用途:** 趣味プロジェクト、プロトタイプ、小規模な内部ツール。

### Pro Tier — $5,000/月

商用利用を想定した本格プラン。

- 投稿: 300,000件/月
- 取得: 1,000,000件/月
- App ID: 3つ
- フィルタードストリーム、フルアーカイブ検索

**Basicとの価格差が25倍**（$200 → $5,000）という異常なジャンプ。この「中間プラン不在」が多くの開発者を苦しめている。

**用途:** ソーシャルリスニングツール、分析プラットフォーム、商用アプリ。

## 2025年10月の大きな変更: Pay-Per-Use Pilot

### 従量課金モデルの導入

2025年10月20日、Xは**Pay-Per-Use Pricing Pilot**を発表した。

**出典:** [X Developers公式発表](https://devcommunity.x.com/t/announcing-the-x-api-pay-per-use-pricing-pilot/250253)

> 「開発者が実際に使用したAPI リクエストに対してのみ支払う、柔軟なクレジットベースのシステム」

### Pay-Per-Useの特徴

| 特徴 | 説明 |
|------|------|
| 従量課金 | 使った分だけ支払い |
| クレジット制 | 事前にクレジットを購入 |
| レート制限緩和 | 月額プランより柔軟 |
| パイロット段階 | 選抜された開発者のみ参加可能 |

### 実際の料金は安くなったのか？

**Social Media Today**の分析によると、必ずしも安くはならない。

Basic相当のアクセス（15,000 Read / 50,000 Write）を従量課金で計算すると、**月額$575相当**になるケースも報告されている。

つまり、**使い方によっては月額プランより高くなる可能性**がある。

### 参加方法

現在はパイロット段階で、申請制。

- 「最良のユースケースと開発者のみが選ばれる」
- 選ばれた開発者には**$500のバウチャー**が付与

## API料金の歴史 — なぜこうなったのか

### Twitter時代（〜2022年）

| 時期 | 出来事 |
|------|--------|
| 2006年 | Twitter API公開。Basic認証、無料で広くアクセス可能 |
| 2010年 | OAuth必須化（「Twitter OAuth Apocalypse」） |
| 2014年 | Gnip買収 → エンタープライズ向けデータ販売開始 |
| 〜2022年 | Standard（無料）/ Premium / Enterprise の3層構造 |

### X時代（2022年〜）

買収後、APIは**収益化の柱**として位置づけられた。

- 無料アクセスの大幅制限
- 料金の大幅値上げ
- AI学習用データ取得への対抗（データアクセス制限）

**結果:** 多くのサードパーティアプリが閉鎖・機能停止。Tweetbot、Twitterrificなど老舗クライアントが相次いで終了。

## サードパーティ代替サービス

公式APIの高コストを避けるため、多くの開発者が代替サービスを利用している。

### 主要な代替サービス比較

| サービス | 1,000ツイート取得 | 特徴 |
|----------|------------------|------|
| **X公式 Pro** | $5.00 | 公式、安定 |
| **TwitterAPI.io** | $0.15 | 非公式、97%安い |
| **Apify Actors** | $0.25〜0.40 | マーケットプレイス型 |
| **自前スクレイピング** | 無料 | 規約違反リスク |

### TwitterAPI.io の例

非公式ながら、**公式の97%安い**という主張。

| 項目 | X公式Pro | TwitterAPI.io |
|------|----------|---------------|
| $5,000で取得可能 | 100万件 | **3,333万件** |
| 1,000件あたり | $5.00 | $0.15 |

**注意点:**
- 非公式サービスは**利用規約違反**のリスク
- サービス停止・データ品質の保証なし
- 商用利用には**法的リスク**を伴う

### xAI API（Grok）

Xが提供する**AI検索API**という選択肢もある。

- リアルタイムのX投稿を検索可能
- 自然言語でのクエリ
- AI要約・分析機能

ただし、こちらも有料であり、用途が限定される。

## ソロビルダー向け 選択肢マトリクス

### ユースケース別推奨

| ユースケース | 推奨プラン | 月額 |
|--------------|------------|------|
| 単純な自動投稿ボット | Free | $0 |
| 趣味プロジェクト（読み取り必要） | Basic | $200 |
| 小規模監視ツール | Basic or Pay-Per-Use | $200〜 |
| 商用ソーシャルリスニング | Pro | $5,000 |
| コスト重視（リスク許容） | サードパーティ | $49〜 |

### 判断フローチャート

```
X連携アプリを作りたい
  │
  ▼
データ取得（Read）が必要？
  │
  ├─ No → Free（$0）で十分
  │
  └─ Yes → 月15,000件で足りる？
              │
              ├─ Yes → Basic（$200/月）
              │
              └─ No → 商用利用？
                        │
                        ├─ Yes → Pro（$5,000/月）
                        │
                        └─ No → サードパーティ検討
                                 （リスク考慮）
```

## 実装時の注意点

### レート制限

| プラン | 15分あたり |
|--------|-----------|
| Free | 50リクエスト |
| Basic | 1,500リクエスト |
| Pro | 3,000リクエスト |

### 認証方式

- **OAuth 2.0** — 推奨
- **OAuth 1.0a** — レガシー対応
- **App-only認証** — 読み取り専用アプリ向け

### API バージョン

- **v2** — 現行の推奨バージョン
- **v1.1** — レガシー（Basic以上で利用可能）

## 今後の展望

### Pay-Per-Use の正式展開

パイロット成功なら、2026年中に正式プランとして追加される可能性。中間価格帯のニーズに応える形になるか注目。

### AI連携の拡大

xAI（Grok）との連携が進めば、従来のAPIとは異なる形でのデータアクセスが可能になるかもしれない。

### 規制動向

EUのデジタルサービス法（DSA）などにより、プラットフォームのAPI提供義務が議論されている。規制によりアクセス条件が変わる可能性も。

## まとめ

| ポイント | 内容 |
|----------|------|
| **Free** | 投稿専用。Read不可 |
| **Basic** | $200/月で最低限のRead |
| **Pro** | $5,000/月で商用レベル |
| **Pay-Per-Use** | 2025年10月〜パイロット |
| **代替** | サードパーティは安いがリスクあり |

**ソロビルダーへの結論:**

読み取りが必要なら**最低$200/月は覚悟**。それが厳しいなら、X連携以外の方法（RSS、公開ページのスクレイピング等）を検討するか、xAI APIなど別のアプローチを探るべき。

公式APIの高コストは当面変わらない見込み。Pay-Per-Use Pilotの動向を注視しつつ、プロジェクトに最適な選択をしよう。
