# ProductHunt 1位獲得の全戦略

**結果:** Daily 1位、1,200+ upvotes、3,000+ サインアップ

ソロ開発者として初めてのProductHuntローンチで、まさかの1位を獲得しました。運ではありません。30日間の準備と、当日の戦略があったからです。

すべてを公開します。

## ローンチしたプロダクト

**概要:** AIを使った議事録自動生成ツール
- Zoom/Google Meet連携
- 自動文字起こし + 要約
- アクションアイテム抽出

**差別化:** 日本語の精度が競合より高い

## なぜProductHuntを選んだか

| 目的 | ProductHuntのメリット |
|------|----------------------|
| **認知獲得** | テック系アーリーアダプターに一気にリーチ |
| **初期ユーザー** | プロダクトに興味がある人が集まる |
| **被リンク** | SEO効果がある |
| **信頼性** | 「PH 1位」のバッジが使える |

## 30日間の準備スケジュール

### Day 1-7: リサーチ

**やったこと:**
1. 過去1年の1位プロダクトを50個分析
2. 共通パターンを抽出
3. 競合のローンチを研究

**見つけたパターン:**

| 要素 | 1位プロダクトの特徴 |
|------|---------------------|
| **タイトル** | シンプル、動詞から始まる |
| **タグライン** | 1文で価値がわかる |
| **動画** | 60秒以内、音声なしでも伝わる |
| **画像** | 5-8枚、使用シーンがわかる |
| **Maker Comment** | ストーリーがある |

### Day 8-14: アセット準備

**作ったもの:**

1. **ロゴ・サムネイル**
   - 240x240px のロゴ
   - 1270x760px のギャラリー画像 x 6

2. **動画（最重要）**
   - 45秒のデモ動画
   - 字幕付き（音声オフでも理解可能）
   - 「問題 → 解決策 → 結果」の構成

3. **ランディングページ**
   - PH用の特別ページを用意
   - 「ProductHuntから来た方へ」のメッセージ

4. **Maker Comment下書き**
   - 開発ストーリー
   - なぜ作ったか
   - 技術的なこだわり

### Day 15-21: コミュニティ構築

**最も重要なフェーズです。**

**Twitter/X での活動:**
```
- 開発過程を毎日投稿
- #buildinpublic タグ活用
- 同じ領域の開発者をフォロー・交流
```

**結果:**
ローンチ前にフォロワー1,200人獲得。

**ProductHuntコミュニティ:**
- 他のプロダクトにupvote・コメント
- 質の高いコメントを心がける
- Maker同士のつながりを作る

### Day 22-28: ハンター探し

**ハンター（Hunter）とは:**
ProductHuntで影響力のあるユーザー。彼らにローンチしてもらうと初速が違う。

**アプローチ方法:**
1. フォロワー1万人以上のハンターをリストアップ
2. 彼らが過去にハントしたプロダクトの傾向を分析
3. 自分のプロダクトと相性が良さそうな人を5人選定
4. Twitter DMで丁寧に依頼

**送ったDM（実際の文面）:**
```
Hi [名前],

I'm launching [プロダクト名] on Product Hunt next week.
It's an AI meeting notes tool with exceptional 
Japanese language support.

I noticed you've hunted similar AI tools before
and thought you might find this interesting.

Would you be open to hunting it?

Here's a quick demo: [Loomリンク]

Thanks for considering!
```

**結果:** 3人中1人がOK。

### Day 29: 最終チェック

**チェックリスト:**
- [ ] 全アセット最終確認
- [ ] ランディングページ動作確認
- [ ] サーバー負荷対策
- [ ] サポート体制確認
- [ ] 当日のスケジュール確定

### Day 30: ローンチ当日

## ローンチ当日のタイムライン

**時間帯の選択:** 
太平洋時間 0:01（日本時間 17:01）

ProductHuntは太平洋時間0時にリセット。丸1日の露出を得るため、0時直後を狙う。

### 0:00-0:30 — 投稿直後

**やったこと:**
1. ハンターがローンチを確認
2. 即座にMaker Commentを投稿
3. Twitter/Xで告知

**Maker Comment（冒頭部分）:**
```
Hi Product Hunt! 👋

I'm [名前], a solo developer from Japan.

For the past 6 months, I've been building [プロダクト名]
because I was frustrated with existing meeting tools
that couldn't understand Japanese properly.

Here's the story...
```

### 0:30-6:00 — 初速を作る

**最初の2時間が勝負。**

**実行したこと:**
1. 準備していた50人にDM/メールで投票依頼
2. 開発者コミュニティ（Discord/Slack）で共有
3. Twitter/Xで3回投稿（時間をずらして）

**注意点:**
- 「投票してください」ではなく「フィードバックください」
- ProductHuntのルールを守る（組織的な投票誘導はNG）

### 6:00-12:00 — 中盤戦

**やったこと:**
1. 全てのコメントに即座に返信
2. 質問には詳細に回答
3. 追加情報を求められたらMaker Commentを更新

**この時点での順位:** 3位

### 12:00-18:00 — 追い上げ

**やったこと:**
1. ニュースレター購読者に告知メール送信
2. LinkedInで投稿
3. 日本のテックコミュニティに共有

**ポイント:**
時間帯によってアクティブな地域が違う。日本・アジア勢が活発になる時間を狙う。

### 18:00-24:00 — 最終盤

**この時点での順位:** 2位（1位と僅差）

**やったこと:**
1. 最後の告知投稿
2. コメント返信を継続
3. リアルタイムで順位をウォッチ

**23:50:** 1位に浮上

**24:00:** 1位でフィニッシュ！

## 成功要因の分析

### 1. 30日間の準備
当日の頑張りより、準備の質が重要。

### 2. コミュニティ構築
事前にフォロワーを作っておいたことで、初速が出た。

### 3. ストーリー性
「ソロ開発者」「日本から」「6ヶ月の開発」というストーリーが共感を得た。

### 4. 全コメント返信
ローンチ日は他の仕事を全て止めて、コメント返信に集中。

### 5. ハンターの力
影響力のあるハンターにローンチしてもらえたことで、初期露出が大きかった。

## やってはいけないこと

1. **組織的な投票依頼** — BANされます
2. **コメント放置** — エンゲージメントが下がる
3. **当日に準備** — 絶対に間に合わない
4. **複数プロダクト同時ローンチ** — 票が分散する
